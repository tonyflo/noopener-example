Example of noopener vs opener vulnerability

The rel attribute of the a tag in HTML can have many values including opener and noopener when used with target="_blank" (which opens the link in a new tab).

The examples below will demonstrate how opener gives the destination page access to the source page's window.opener object which can be exploited in malicious ways.

1. opener

When you click on this link, the new page will have access to this page's window.opener object and change its content.

<a href="page.html" target="_blank" rel="opener">this link</a>

2. noopener

When you click on this link, the new page will NOT have access to this page's window.opener object.

<a href="page.html" target="_blank" rel="noopener">this link</a>

3. Default behavior

When you click on this link, most modern browsers will default to the noopener behavior.

<a href="page.html" target="_blank">this link</a>
